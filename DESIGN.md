# Lucian Blog Design

> 版本：1.0  
> 状态：当前设计基线（Design Baseline）  
> 适用范围：首页、文章页、普通页面、投资练习页  
> 最后更新：2026-07-13

## 1. 文档目的

这份文档定义 Lucian Blog 的设计方向、视觉约束和验收方法。它是后续页面迭代的设计基线，而不是一次性的风格说明。

所有新页面和组件都应先满足本文件，再考虑增加功能。若实现与本文件冲突，应优先讨论并更新设计决策，不能在单个页面中静默引入例外。

## 2. 博客定位

Lucian Blog 是一份持续更新的个人思考档案，记录三类内容：

1. 对会赚钱产品的拆解；
2. 对个人投资纪律的练习；
3. 对自身认知变化的记录。

设计需要传达三个特征：

- **克制**：页面服务于内容，不用装饰制造注意力；
- **诚实**：保留判断形成和被修正的过程；
- **长期**：视觉不过度依赖短期潮流，适合多年持续写作。

设计隐喻是“安静的编辑手记”：接近纸张、墨水和批注，而不是产品后台、资讯门户或金融仪表盘。

## 3. 设计目标与非目标

### 3.1 设计目标

- 让标题、摘要和正文形成清楚的阅读顺序；
- 用大面积留白降低视觉噪声；
- 让产品拆解和认知文章可以长期扩展；
- 让投资练习的数据易于比较，但不过度金融化；
- 在桌面和手机上保持相同的安静感；
- 不依赖 JavaScript 也能完成主要阅读和导航。

### 3.2 非目标

本博客不追求：

- 新闻门户式的信息密度；
- SaaS Dashboard 式的卡片堆叠；
- 复杂的数据可视化；
- 依靠动画、渐变或图片建立视觉冲击；
- 把所有内容都放入顶部导航；
- 用增长指标替代真实内容。

## 4. 核心设计原则

### 4.1 内容先于界面

页面的第一视觉焦点必须是内容命题，而不是网站功能。首页用一句核心陈述建立主题，文章页直接展示标题和正文，投资页首先解释练习规则。

新增视觉元素前，应先回答：它是否帮助读者理解内容、判断层级或完成导航？如果答案是否定的，就不应加入。

### 4.2 留白是主要视觉元素

留白不是“没有设计”，而是用于建立节奏、层级和思考空间。空白区域应明显大于文字、边框和数据的视觉占用。

桌面端关键视口采用以下验收规则：

- 首屏主要内容的视觉包围区域不超过视口面积的 **38.2%**；
- 因此，首屏感知留白应超过 **61.8%**；
- 首页 Hero 至少占据一个视口高度；
- 内容分区上下留白总和不低于 `24vh`；
- 文章和普通页面正文宽度不超过 `680px`；
- 不使用大面积图片、实色卡片或背景图填满空白。

这里的 61.8% 是关键视口的视觉密度目标，不是要求计算每个字形像素。验收时应在 `1440 × 900` 桌面视口中检查标题、介绍和操作提示形成的整体包围区域。

移动端宽度有限，不机械套用 61.8% 面积规则。移动端优先保证文字可读，同时保持以下节奏：

- Hero 仍接近一个视口高度；
- 内容分区上下留白总和不低于 `24vh`；
- 页面左右边距保持 `16px`；
- 不为了增加空白而把正文缩小到 16px 以下。

### 4.3 少即是约束，而不是装饰风格

极简设计要求减少决策数量：更少的颜色、更少的字体角色、更少的导航、更少的边框和更少的动画。

当页面显得拥挤时，优先采取以下顺序：

1. 删除不必要元素；
2. 合并重复信息；
3. 增加间距；
4. 调整字体层级；
5. 最后才考虑新增组件。

### 4.4 层级依靠排版，不依靠颜色堆叠

标题、正文、注释和数据之间的层级主要由字号、字重、间距和位置建立。强调色只用于少量定位元素，不能把多个区块涂成不同颜色。

### 4.5 数据表达保持克制

投资页面用于记录纪律，不用于营造交易情绪。收益变化必须同时通过正负号和数字表达，不能只使用红色或绿色。

## 5. 视觉语言

### 5.1 色彩系统

全站最多使用三种基础颜色：

| Token | 色值 | 用途 |
|---|---:|---|
| `--paper` | `#F4F1EA` | 页面背景、卡片内部和反色文字 |
| `--ink` | `#191917` | 正文、标题、边框基础色 |
| `--accent` | `#A6452D` | 小标题、焦点、链接提示和品牌句点 |

颜色比例建议：

- 纸张色约占页面可见面积的 `90%+`；
- 墨色用于全部主要信息；
- 朱砂色只用于少量定位和反馈，通常不超过可见面积的 `3%`。

允许通过透明度或 `color-mix()` 从这三种颜色生成弱化文本和分隔线，例如：

```css
--line: color-mix(in srgb, var(--ink) 18%, transparent);
--line-soft: color-mix(in srgb, var(--ink) 9%, transparent);
```

不允许新增第四种品牌色。状态信息也应使用文字、符号和结构表达，而不是引入绿色、蓝色或黄色。

### 5.2 字体系统

字体分为两个角色。

**衬线字体（Serif）**用于：

- Hero 标题；
- 页面和文章标题；
- 正文；
- 文章列表标题；
- 关键投资数字。

```css
--serif: "Iowan Old Style", "Baskerville",
         "Noto Serif SC", "Source Han Serif SC", serif;
```

**无衬线字体（Sans Serif）**用于：

- 导航；
- 日期和元数据；
- 小型标签；
- 表头和更新说明。

```css
--sans: "Avenir Next", "Noto Sans SC",
        "PingFang SC", sans-serif;
```

字体角色不能随页面改变。不要在某个新页面中单独引入第三套字体。

### 5.3 字号层级

| 层级 | 桌面端 | 移动端/自适应 | 说明 |
|---|---:|---:|---|
| Hero 标题 | `42–78px` | `42px` 起 | `clamp(42px, 6vw, 78px)` |
| 页面标题 | `40–68px` | `40px` 起 | `clamp(40px, 6vw, 68px)` |
| 分区标题 | `32–52px` | `32px` 起 | `clamp(32px, 4vw, 52px)` |
| 正文二级标题 | `28px` | `28px` | 保持明显章节边界 |
| 列表文章标题 | `22px` | `22px` | 不使用卡片标题样式 |
| 正文 | `17px` | `16px` | 行高约 `1.78` |
| 摘要 | `14–19px` | 同比例 | 根据页面语义选择 |
| 元数据 | `10–13px` | 同比例 | 必须保持可辨认 |

中文正文不应小于 `16px`。长文本行高应保持在 `1.7–1.8`，以支持连续阅读。

### 5.4 字间距与数字

- 大标题允许轻微负字距，以形成紧凑的编辑感；
- 小标签使用较大的正字距，形成定位标记；
- 日期、股价、投入和收益率使用等宽数字 `font-variant-numeric: tabular-nums`；
- 正文不使用人为增加的字间距。

### 5.5 边框与层次

边框只用于建立结构，不用于装饰：

- 分区线：墨色 `9%` 透明度；
- 列表和表格线：墨色 `18%` 透明度；
- 引用标记：`2px` 朱砂色左边框；
- 不使用阴影；
- 不使用大圆角；
- 不使用玻璃、模糊或渐变效果。

## 6. 布局系统

### 6.1 全局外壳

```css
.site-shell {
  width: min(100% - 48px, 1120px);
  margin: 0 auto;
}
```

桌面端最大宽度为 `1120px`，左右至少保留 `24px`。移动端左右边距为 `16px`。

### 6.2 阅读宽度

- 文章和普通页面：最大 `680px`；
- Hero 标题区域：最大 `760px`；
- 首页文章分区：最大 `860px`；
- 投资表格：最大 `920px`；
- 文章摘要：最大 `560px`。

不同宽度服务于不同信息类型，但必须保持左侧基线和清楚的阅读关系。

### 6.3 间距节奏

基础间距遵循 4/8 像素节奏。常用层级为：

```text
8 / 12 / 16 / 24 / 32 / 40 / 48 / 64 / 72 / 80 / 96
```

大区块使用视口单位建立留白：

- Hero：桌面 `18vh` 上、`24vh` 下；
- 首页分区：桌面 `18vh` 上、`12vh` 下；
- 文章与页面：桌面 `18vh` 上、`20vh` 下；
- 移动端分区：`14vh` 上、`10vh` 下。

不要为了填满页面而缩小这些大间距。

### 6.4 响应式断点

当前主要断点为：

```css
@media (max-width: 700px)
```

移动端变化包括：

- Header 高度由 `96px` 调整为 `80px`；
- 首页分区从两列标题结构变为单列；
- 文章列表日期移动到标题上方；
- 投资汇总从三列变为三行；
- 投资表格在独立容器内横向滚动；
- 页面本身不能出现横向滚动。

## 7. 页面结构

### 7.1 全局页面骨架

```text
┌──────────────────────────────────────────────┐
│ Lucian.                         产品   认知   │
├──────────────────────────────────────────────┤
│                                              │
│                  MAIN                        │
│                                              │
├──────────────────────────────────────────────┤
│ 持续记录，诚实修正。       GitHub · 练习     │
└──────────────────────────────────────────────┘
```

Header 和 Footer 保持稳定，不因页面类型改变位置或视觉语言。

### 7.2 首页

首页负责说明博客主题和展示两类公开内容。

```text
Header

[LUCIAN / NOTES]
记录产品、资本与认知的缓慢复利。
一句解释
向下阅读

01  会赚钱的产品
    文章标题 / 摘要 / 日期

02  认知迭代
    文章标题 / 摘要 / 日期

Footer
```

规则：

- 首屏只能有一个主标题；
- Hero 不加入头像、插画、按钮组或社交数据；
- 产品拆解必须先于认知迭代；
- 文章列表使用分隔线，不使用卡片背景；
- 每条记录只展示标题、摘要和日期；
- 空栏目显示简短空状态，不展示营销式 CTA。

### 7.3 文章页

文章页用于连续阅读。

```text
返回首页
内容类型标签
文章标题
日期 / 字数

正文
```

规则：

- 正文最大宽度 `680px`；
- 每个段落只表达一个主要信息；
- 二级标题与前文保留明显间距；
- 引用使用朱砂色左边框，不使用彩色引用卡；
- 图片若以后加入，应有明确内容作用、说明文字和替代文本；
- 不增加自动播放、浮动分享条或阅读进度动画。

### 7.4 投资练习页

投资页是弱隐藏页面。它不出现在顶部导航和首页内容栏目中，只在 Footer 的社交链接右侧提供入口：

```text
GitHub · 练习
```

页面结构：

```text
PAGE
一元练习
练习说明

累计投入 | 当前市值 | 总收益率
更新时间与免责声明

公司 | 当前股价 | 累计投入 | 当前市值 | 收益率

计算方式
```

规则：

- 页面设置 `noindex, nofollow`；
- 弱隐藏不等于权限保护，知道 URL 的人仍可访问；
- 汇总数字使用衬线字体和等宽数字；
- 表格每行表示一家公司；
- 数字默认右对齐，公司名称左对齐；
- 正收益显示 `+`，负收益显示 `−`；
- 不使用红涨绿跌作为唯一信息；
- 移动端只允许表格容器横向滚动，整个页面不能横向溢出；
- 页面必须显示数据更新时间和“非投资建议”说明。

投资计算保持透明：

```text
累计份额 = Σ（每月投入 ¥1 ÷ 当月人民币等值股价）
当前市值 = 累计份额 × 最新一期人民币等值股价
收益率 =（当前市值 − 累计投入）÷ 累计投入 × 100%
```

## 8. 导航与信息可见性

### 8.1 顶部导航

顶部只保留：

- 品牌入口 `Lucian.`；
- 产品；
- 认知。

不把投资练习、RSS、关于页面或社交链接加入顶部。顶部导航的职责是帮助读者理解公开内容结构，不是列出全部 URL。

### 8.2 页尾导航

页尾承载低优先级信息：

- 一句长期写作原则；
- 社交媒体链接；
- 投资练习入口。

投资练习入口必须紧跟社交链接，并用中点分隔。若以后增加多个社交媒体链接，投资入口仍位于所有社交链接之后。

## 9. 交互与动画

交互应安静、快速且可预测。

- 默认过渡时间：`220ms`；
- 链接 Hover：文字转为朱砂色；
- 文章列表 Hover：只允许轻微左移或改变文字颜色；
- “向下阅读”提示：箭头最多下移 `4px`；
- 不使用页面转场、视差、粒子、滚动劫持或循环动画；
- 所有主要内容在动画执行前就必须可读。

用户设置减少动态效果时：

```css
@media (prefers-reduced-motion: reduce) {
  html { scroll-behavior: auto; }
  * { transition-duration: 0.01ms !important; }
}
```

## 10. 无障碍要求

所有迭代必须满足以下要求：

- 每页只能有一个主要 `h1`；
- 标题层级不能跳过；
- 页面包含“跳到正文”链接；
- 键盘焦点使用 `2px` 朱砂色可见轮廓；
- 所有可点击目标至少 `44 × 44px`；
- 正文和背景对比度达到 WCAG AA；
- 链接不能只依靠颜色识别，正文链接保留下划线；
- 日期使用语义化 `<time>`；
- 表格必须使用 `<th>`、`scope` 和清楚的列名；
- 表格滚动区域可通过键盘聚焦；
- 图标若以后加入，应使用 SVG 并提供可访问名称；
- 装饰性符号必须设置 `aria-hidden="true"`；
- 页面缩放后不能隐藏正文或禁用浏览器缩放。

## 11. 性能要求

当前设计默认不需要客户端 JavaScript。

- 不为纯装饰引入 JavaScript；
- 不加载远程 Web Font，优先使用本地字体栈；
- 首屏不使用大图或视频；
- CSS 保持单一主文件；
- 不引入组件框架解决单个页面问题；
- 新图片优先使用 AVIF 或 WebP，并声明尺寸；
- 非首屏图片使用懒加载；
- 页面构建后不能出现阻塞正文阅读的第三方请求。

## 12. 内容模型

### 12.1 产品拆解

```yaml
layout: post
category: product
title: 文章标题
description: 首页的一句话摘要
date: YYYY-MM-DD HH:MM:SS +0800
```

### 12.2 认知迭代

```yaml
layout: post
category: cognition
title: 文章标题
description: 首页的一句话摘要
date: YYYY-MM-DD HH:MM:SS +0800
```

### 12.3 投资数据

```yaml
monthly_contribution: 1
updated_at: YYYY-MM-DD
companies:
  - name: 公司名称
    ticker: TICKER
    snapshots:
      - date: YYYY-MM-01
        price_cny: 100.00
```

产品和认知文章公开展示。投资数据只在弱隐藏页面展示。

## 13. 禁止模式

后续设计中不要引入以下模式：

- 超过三种基础颜色；
- 紫色或多色渐变；
- 大量卡片和阴影；
- 玻璃拟态、黏土拟态或新拟态；
- 结构性 Emoji；
- 自动轮播和弹窗；
- 固定悬浮的分享工具；
- 红绿作为收益的唯一表达；
- 为填满页面而增加无意义插画；
- 多级复杂导航；
- 在首页直接展示投资数据；
- 移动端页面级横向滚动；
- 无法通过键盘访问的交互；
- 只在 Hover 时才出现的重要信息。

## 14. 页面迭代流程

每次页面迭代按以下顺序进行：

1. **明确内容任务**：读者在该页面需要理解什么；
2. **建立信息层级**：先确定标题、正文、注释和操作顺序；
3. **检查是否可以删除**：删除不支持主要任务的元素；
4. **复用现有 Token**：颜色、字体和间距优先使用现有系统；
5. **实现桌面结构**：在 `1440 × 900` 检查留白和阅读宽度；
6. **实现移动结构**：在 `375 × 812` 检查换行和溢出；
7. **完成可访问性检查**：键盘、标题、对比度、语义标签；
8. **完成构建与链接检查**：确认 Jekyll 和内部链接正常；
9. **再做视觉审查**：确认没有因功能增加而破坏克制感。

如果新需求需要违反本文件，应先记录：

- 为什么现有规则无法满足需求；
- 违反哪一条规则；
- 是否存在更简单方案；
- 这个例外是否应成为全局规则。

## 15. 验收清单

### 15.1 视觉

- [ ] 基础颜色不超过 3 种；
- [ ] 桌面首屏感知留白超过 61.8%；
- [ ] 正文宽度不超过 680px；
- [ ] 没有阴影、渐变和无意义背景图；
- [ ] 朱砂色只用于少量强调；
- [ ] 字体角色与当前系统一致；
- [ ] 页面仍然像编辑手记，而不是 Dashboard。

### 15.2 内容和导航

- [ ] 首页只有产品拆解与认知迭代两个公开栏目；
- [ ] 每篇文章被放入正确分类；
- [ ] 投资入口不出现在顶部导航；
- [ ] 投资入口位于社交链接右侧；
- [ ] 投资页包含更新时间和免责声明。

### 15.3 响应式
